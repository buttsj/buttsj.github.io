Super Mario World in C#

For my Junior year project course I was asked to design a 2D platformer. Though this was the overarching project, the main goal of the class was to understand the Agile methodology and create readable code.


I took the role of Team Lead for my group and together we designed a replica of the first level of Super Mario World. The game ran through Visual Studio and XNA Game Studio and was programmed in C#.


The final sprint of the project was to add a feature to the game that was not in the original, and my team chose a few. The first was a level following the style of the video game VVVVVV where the player can control gravity for the character. The second was in-game achievements that appear similar to Xbox achievements. Lastly we created a new Mario power-up that turned him into a ninja with throwing stars.

  • Class: CSE 3902
  • Date: Spring 2015
  • Skill area(s): Programming/Game Development

Zombie Escape: GameShow

I picked up the hobby of designing maps for video games after the decades of gaming I have exposed myself to. I published a few maps on the modding site GameBanana, but one that I took particular pride in was called GameShow.


The map was designed for the game Counter-Strike: Source. It was used in the custom gamemode called Zombie Escape. The premise of this gamemode is to escape the level without being infected by the zombies. The maps for this gamemode varied greatly and a few were technical feats.


A few smart developers even created boss fights within this gamemode and it became something I had to try. My map was themed around three separate "escapes" that had to all be beaten in order to fight the final boss.


The map took months of designing and testing before I felt it was ready to be released, and even then I got lots of feedback on how to make it better.


At the time of this writing my map has a 9.67/10 rating on GameBanana with 6800+ views and 3000+ individual downloads (not including unrecorded downloads from servers).

  • Software used: Source SDK, Counter-Strike: Source
  • Date: Summer 2012
  • Skill area(s): Game Design

Custom Shell

For my Introduction to Operating Systems course I was asked to design a simple shell using C programming. The shell accepted user commands that were executed in a child process.


The first part of this assignment was to implement a history command and a few custom commands, like "Run Most Recent (rr)" and "Run Command # (r num)." The max history can also be set with "sethistory #".


The second part of the assignment was to enhance the history feature by making it persistent between sessions. This was done through saving the history with each session to the user's Home directory. I also implemented the ability to change the current directory with the "cd" command.

  • Class: CSE 2431
  • Software used: Linux, C programming
  • Date: Fall 2015
  • Skill area(s): Programming

Core Interpreter written in Java

This was an ongoing project in my Principles of Programming Languages course at The Ohio State University. In this course I designed an Interpreter for the Core Language.


My interpreter made use of a Tokenizer to build a list of tokens for the language. My Tokenizer used a Singleton pattern to make it accessable and consistent to all objects.


The interpreter itself was designed using an Object-Oriented implementation, so all of the Backus-Naur Form non-terminal symbols represented an object. The interpreter had three parts:


  1. Parser: create all of the objects needed for the code provided, error check as well.
  2. Printer: pretty-print the code for the user.
  3. Executor: run through the code and perform the operations and provide any output.
  • Software used: Eclipse, Java
  • Date: Spring 2016
  • Skill area(s): Programming, Computer Science

Unity3D Course Projects

Lab 01 focused on learning the basic functions in Unity, including transformations and building scenes.

Lab 02 introduced rigid bodies and how to simulate these in Unity. This lab taught the formulas for performing rigid body operations on a Bunny model.

Lab 03 worked with explicit and implicit mass-spring systems to simulate cloth and how objects interact with it.

Lab 04 was a simple shallow wave simulator on a water surface. For extra work, I implemented a block that reflected waves.

Lab 05 taught how Unity deals with animations, specifically on a human character. This included running, jumping, and many more animations.

Finally, the final project was to take any techniques learned in the course and implement them in a personal game.


I chose to create a Table-Top Simulator, and in it I made the board game Settlers of Catan. I made use of rigid bodies for game pieces to simulate things like dice rolling.

  • Date: Spring 2016
